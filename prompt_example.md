# AI 개발 프롬프트 작성 유의 사항

AI에게 일을 시키는 것은 사람에게 일을 맡기는 것과 매우 비슷하다.  
내가 잘 모르는 복잡한 일을 막연하게 시키면, AI도 제대로 수행하기 어렵고,  
결과가 올바른지 검증하기도 힘들다.  
따라서 **검증 가능한 형태로 단계적으로 일을 시키는 것**이 중요하다.

## 1. 항상 "검증 가능한 단위"로 일을 시킨다
- 한 번에 큰 작업을 맡기지 않는다.
- **중간 결과를 확인할 수 있는 작은 단계**로 나눠서 지시한다.
- 각 단계에서:
  - AI가 수행할 작업
  - 그 단계의 산출물(GAME_RULE.md, 계획서, 코드 초안 등)
  - 내가 어떤 기준으로 “잘 됐다/안 됐다”를 평가할지  
  를 명확히 해야 한다.

이렇게 해야 문제를 **초기 단계에서 빠르게 발견하고 수정**할 수 있다.

## 2. 방법 A – 계획을 먼저 최대한 구체적으로 나눈다
처음부터 코드를 만들라고 하면 오류가 많아지므로  
AI에게 다음을 먼저 요청한다:

- 템플릿 구조 분석
- 기능 분해
- GAME_RULE.md 정리
- 어떤 파일을 어떤 순서로 수정할지 계획 수립
- 단계별 git 업데이트 계획

즉, **“계획 → 설계 → 구현 → 테스트”** 순서로 AI를 움직이게 하는 것이 핵심이다.

## 3. 방법 B – 껍데기(스켈레톤)를 먼저 만들고 내부를 채운다
어떤 경우에는 결과물을 빨리 보고 싶을 수 있다.  
이럴 때는:

1) “껍데기만 있는 초기 버전”을 먼저 만들어 달라고 하고  
2) 이후 단계적으로 내부 로직을 구체화하는 방식을 쓴다.

이 방식은 특히:
- 게임 UI가 처음부터 보여야 할 때  
- 학생들과 함께 빠르게 결과를 확인해야 할 때  
효과적이다.

# AI 바이브 코딩 기반 포즈 게임 개발 프롬프트

아래는 위 원칙을 기반으로 작성된 **단계적 개발 프롬프트 예시**이다.  
이 흐름은 Teachable Machine 모델 학습과 AI 코딩을 결합한 전체 개발 과정이다.

---

# 1) 템플릿 구조 인식

우선 README를 읽고 tm-pose-template 구조를 분석해줘.  
어떤 파일이 무엇을 담당하는지 요약하고,  
이번 게임 제작을 위해 어떤 부분을 수정해야 하는지 계획을 세워줘.

---

# 2) 게임 설명 및 GAME_RULE.md 작성

게임 이름: **"Catch Zone"**

기본 구조:
- 화면은 좌(LEFT), 가운데(CENTER), 우(RIGHT) 3개 구역으로 구성된다.
- 플레이어는 포즈 인식을 이용해 바구니를 움직인다.
  - 왼쪽 포즈 → LEFT  
  - 중립 포즈 → CENTER  
  - 오른쪽 포즈 → RIGHT  

아이템 종류:
1. 폭탄(Bomb) → 닿으면 즉시 게임 오버  
2. 사과(Apple) → +100점  
3. 배(Pear) → +150점  
4. 오렌지(Orange) → +200점  

게임 진행 규칙:
- 아이템은 화면 상단에서 3개 구역 중 하나로 떨어진다.
- 바구니와 같은 구역일 때만 아이템 획득.
- 과일 획득 시 점수 증가.
- 폭탄 획득 시 즉시 게임 오버.

놓침(미스) 규칙:
- 과일 놓침 1번 → 경고  
- 과일 놓침 2번 → 게임 오버  

단계(Level) 시스템:
- 각 단계는 20초 동안 진행.
- 1단계 drop time = 2초  
- 단계가 오를 때마다 drop time 0.2초씩 감소
- 아이템 생성 간격 = drop time의 60% ~ 80%

게임 오버 없이 20초가 지나면 자동으로 다음 단계로 진행된다.

위 내용을 기반으로 **GAME_RULE.md** 파일을 만들어줘.  

- 개요  
- 구역 정의  
- 조작 방식  
- 아이템 종류  
- 점수 규칙  
- 게임 오버 조건  
- 레벨 시스템  
- UI 요소  

등을 체계적으로 정리해줘.

완료되면:  
“GAME_RULE.md 작성 완료”라고 답해줘.

# 3) Git 업데이트

지금까지 개발한 내용을 git에 업데이트해줘.

---

# 4) Teachable Machine 포즈 모델 학습 대기

이제 내가 포즈 모델을 Teachable Machine에서 학습해올게.  
모델 업로드할 때까지 기다렸다가 다음 단계로 넘어가자.

---

# 5) 모델 학습 후 업데이트

학습한 포즈 모델을 업데이트했어.
업데이트한 모델이 잘 만들어졌는지 확인해줘.

---

# 6) 개발 시작

이제 본격 개발을 시작해줘.

1. 템플릿의 파일 구조에 맞게  
2. GAME_RULE.md 규칙을 정확히 반영하여
3. 게임이 기본적으로 동작하는 버전을 완성해줘.

가능하면 한국어 주석을 자세히 달아줘.

---

# 7) 로컬 테스트 후 수정

내가 로컬에서 테스트하며 문제점을 알려줄게.  
예:

```

바구니가 오른쪽 포즈에서 너무 느리게 이동해. 속도를 조금 올려줘.

```

이런 식으로 내가 오류/수정 요청을 보내면  
문제된 부분만 골라서 수정한 코드 또는 수정된 함수만 보여줘.
